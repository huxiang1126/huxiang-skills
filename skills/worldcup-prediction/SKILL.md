# worldcup-prediction: 世界杯赛前盘口预测总控

一条命令完成：球队解析 -> 内部回溯账本读取 -> 最新资料检索 -> 首发/天气/盘口核查 -> 无盘口独立建模 -> 多 Agent 交叉判断 -> 盘口价值/诱盘核查 -> 正路/逆路校准 -> 单位化投注执行建议 -> 球迷可读结论 -> 两张朋友圈图 -> 本地预测账本记录。

核心原则：

- 先检索，后结论。
- 查不到就写「暂无可靠资料」。
- 不编盘口、不编伤停、不编天气。
- 18 个 Agent 不是装饰词。只要当前环境支持 sub-agent / Research tool，就必须真实派出 18 个独立证据任务；如果受并发限制，分批跑完 18 个。不能把一次综合搜索包装成 18 个 Agent。
- 如果当前环境不支持真实 sub-agent，必须明说「当前环境不支持真实 18-Agent 并行，改用定向检索模拟」，并且不得写「18 个 Agent 已完成」。
- 每个 Agent 必须留下可审计回执：资料来源、检索时间、关键发现、暂无可靠资料项、对盘口判断的影响。没有回执，就不能算该 Agent 完成。
- 必须给出一个唯一主推盘口；默认要在让球盘或大小球里给出「最优可执行方向」。只有触发不投注熔断条件时，才允许写「本场不建议投注」。
- 盘口不是预知赛果，盘口是市场价格和风险管理信号；必须识别过热、偏浅、偏深和疑似诱盘，但不能阴谋论化。
- 盘口不能当答案。必须先给出「不看盘口时的独立模型倾向」，再和盘口对照。
- 每一场必须先建立「本场独立基准」：只用本场球队、首发、战术、天气、赛程、盘口价格和市场热度来形成初始概率。回溯账本只能提出审计问题，不能直接改写本场基准概率。
- 禁止赛果迁移惯性：上一场强队没穿盘，不代表下一场强队不能穿；上一场强队穿盘，不代表下一场可以盲追；上一场刚推小球，也不代表下一场必须避开或继续推小球。
- 禁止低比分默认锚点：不能为了显得稳，就把主比分长期压在 `1:0 / 2:0 / 2:1`。每场必须独立判断比赛是否会被早进球、边路爆点、高点冲击、定位球、替补火力、弱队崩盘或追分空间打开到 3 球以上。
- 大球不是波胆。波胆是单个比分，命中率天然低；大球是多个比分路径的叠加，例如 `2:1 / 3:0 / 3:1 / 4:0 / 4:1` 都可能支持大球。禁止把「波胆低概率」误用成「大球不值得推」。
- 低位防守不自动等于小球。必须检查强队是否有破密集防守的工具：边路传中、高点、二点球、远射、定位球、肋部直塞、替补冲击；如果这些工具成立，弱队摆大巴也可能被打成大球或深盘穿盘。
- 不能默认正路。最终可以顺盘，也可以逆盘，也可以不碰；关键是价格和比赛逻辑是否匹配。
- 如果最终顺着市场热门方向，必须额外说明「为什么这次顺盘仍然有价值」。
- 不能默认反路。上盘价格偏贵，只能说明上盘价值下降，不能自动说明下盘有价值；下盘必须用自己的不败/赢盘概率、比分路径、首发天气战术和赔率补偿单独通过校验。
- 反热门不是目的，找价格错误才是目的。热门可以穿盘，冷门也可以被高估；每场必须实事求是判断：该谁赢就谁赢，该谁穿盘就谁穿盘，该谁打不穿就打不穿。
- 庄家思路、球迷思路、热门资金和冷门补偿都只能作为价格信号，不能替代足球判断。最终主推必须同时通过「比赛事实」和「盘口价格」两道门。
- 存在「最优投注建议」，但不是每场都有。最优不等于最刺激，也不等于波胆；它必须同时满足方向清楚、价格未坏、风险可解释、资金效率更好。若盘口优势很薄但仍有一个通道略优，必须给出低单位「谨慎可跟 / 可博」建议；只有所有通道都无法通过最低可执行阈值，才允许不投注。
- 「最优投注建议」必须来自让球盘、大小球、不投注三通道的强制仲裁，不能默认从让球盘里选。让球盘只是一个投注通道，不是默认主通道。
- 「不投注」是熔断选项，不是默认保守答案。博彩公司定价本来就会吸收强弱、伤停、天气、公众情绪和利润水位；我们不能因为这些因素被盘口计入，就机械得出“不划算”。总控必须判断：在庄家希望市场下注的价格里，哪个通道仍然最接近可执行。
- 如果连续 2 场主推为「不投注 0U」，下一场必须触发「可执行建议优先复核」：先把让球盘和大小球里最接近可投的一项找出来；除非触发明确熔断条件，否则必须给出一个低单位主推，而不是第三次继续不投注。
- 如果最近 5 场赛前预测里有 4 场及以上主推同一盘口类型，下一场必须触发「通道偏置复核」：先证明为什么大小球或不投注不如当前主推，再允许继续选择同一类型。
- 「胜负倾向」和「投注主推」必须分离。胜负倾向只回答 90 分钟赛果；投注主推必须明确是让球盘、大小球，还是不碰。
- 「投注主推」和「波胆」必须分离。让球盘/大小球才是主执行通道；波胆只能是小额附加，不得喧宾夺主。
- 默认使用单位制 `U` 给执行建议，不输出具体金额，除非用户明确提供预算或单位金额。`1U` 表示用户自己预先设定的「单场主推基准娱乐单位」，不是固定金额，也不是模型替用户决定的金额；如果用户习惯按比例管理，才可把 `1U` 理解成总娱乐预算里很小的一份，例如 1%-2% 上限。
- 只要最终输出、朋友圈文案或图 B 出现 `U`，必须同时给一句解释：`U=自定娱乐单位，不是固定金额；0.05U=主推1U的5%小额观点票`。图 A 空间不足时可写 `1U=自定单位`。
- 波胆投注比例必须明显小于让球盘/大小球主推：单个波胆建议不超过主推投入的 5%-10%，单场全部波胆合计不超过主推投入的 20%。比分分散、盘口更优、赔率不划算时，必须写「不建议波胆」。
- 最多给 1-2 个可选波胆比分；禁止为了覆盖焦虑列 3 个以上波胆下注方向。三个预测比分是分析路径，不等于三个都要下注。
- 盘口覆盖路径和波胆建议必须分开。让球盘覆盖的比分只是说明主推盘口在这些赛果下如何结算，不等于这些比分都值得买波胆。若出现「盘口覆盖 2:1 / 0:1」但「波胆不追加 2:1 / 0:1」，必须解释为：这些是盘口结算路径，不是波胆下注建议。
- 禁止追损、翻倍、加仓追回、连续黑后扩大单位。所有投注执行建议必须限定在娱乐预算内。
- 禁止只写「看好 {球队}」作为海报或最终投注信息。必须写成「90分钟倾向：{球队}胜 / 平」和「主推投注：{盘口类型} {方向}」。
- 如果主推是让球盘，必须把盘口符号翻译成球迷能直接理解的规则：赢盘、输盘、走水、赢半、输半分别在什么比分条件下发生。
- 禁止只写「不追」「不追高」「降盘不追」作为风控说明。必须写成「盘口失效条件」，并说明触发后是原主推推荐作废，不代表自动改买反方向；需要重新检索盘口和首发后再判断。
- 每次赛前预测前必须读取本地回溯账本，检查最近错因；每次赛后复盘必须把结果、盘口命中、比分命中和错因标签写入账本。
- 回溯账本和错因标签是内部模型语言，不能原样展示给普通球迷。最终输出和图片里必须翻译成「赛前证据校准」「临场修正」「风险提醒」这类可信、人话表达。
- 回溯校准不是下注经验迁移。只有当「本场自身证据」也出现同类脆弱点时，才允许降低强度；不能因为上一场红黑、某类盘口刚输/刚赢，就自动改变本场主推通道。
- 天气不能只列数据。必须说明它对进球数、比赛节奏、体能、逼抢、射门质量、门将处理球或定位球的具体影响。
- 若距开球约 90 分钟内，必须优先抓取官方/权威媒体确认首发；若还没有确认首发，必须标注「预测首发」，并说明首发不确定性对盘口强度的影响。
- 预测海报必须加强视觉冲击：允许毛笔字/手写冲击字、英文功能小字、强对比构图，但仍要清楚写出主推盘口和赢盘条件。
- 预测海报不能只是信息卡。必须出现两队核心球星/代表性球员或可识别的球员剪影，并为两队各配置一个国家队视觉符号，例如荷兰狮子、日本红日、德国鹰、法国高卢鸡、阿根廷太阳纹、英格兰三狮意象等；不确定时使用国旗几何、队服色和城市球场元素，不能自编官方 logo。
- 图 A 顶部主标题必须有毛笔字/手写冲击字气势；若生成结果变成普通黑体、PPT 标题或文字缺字，必须再生成一版更少字、更强主标题的海报。
- 分析思路长图必须继承已验证的 `huxiang-card` 字体和版式母版；中文正文/标题优先走 `KingHwa_OldSong`（京華老宋体）渲染路径，不能临时改成系统 UI 字体或自造 dashboard 风格。

## 硬规则

1. 必须联网检索最新资料。优先官方、权威体育媒体、球队官方、可靠数据/盘口平台。
2. 任何具体数字必须有来源：赛程、比分、近况、伤停、盘口、赔率、天气。
3. 盘口必须写检索时间；超过 24 小时的盘口要标注「可能已变化」。
4. 找不到可靠盘口，写「暂无可靠盘口数据」，不要自己估。
5. 不输出完整 Agent 长篇过程，只输出固定结果。
6. 不使用「梭哈」「稳赚」「必中」「重注」等刺激下注词。
7. 必须提醒：赛前个人分析，仅供娱乐参考，不保证赛果。
8. 图片不能出现 FIFA 官方 logo、投注平台品牌、赌场元素。
9. 禁止把盘口当作结论来源。先做无盘口判断，再做盘口对照。
10. 禁止为了显得稳而永远选热门方、让球方、小球方。足球是低得分高波动项目，必须保留逆路可能。
10.1. 禁止为了显得专业而永远压低比分。若强队具备持续进球工具、弱队落后后会被迫压出、或双方都有进球路径，必须允许主推大球、允许主比分或次选比分进入 3-5 球区间。
11. 禁止使用模糊投注词：`看好加拿大`、`看好阿根廷`、`站法国` 不能单独出现，必须标注它是「90分钟胜负倾向」还是「让球盘投注方向」。
12. 海报里最醒目的主信息必须是「主推投注」，不是胜负倾向；如果主推大小球，就不能把球队名做成最大结论。
13. 主推让球盘时，禁止只写 `{球队} -0.5` 或 `{球队} +0.25`。必须同时写「让胜/受让」和赢盘规则。
14. 「盘口失效条件」必须明确回答三件事：哪个盘口变化会让原推荐失效、为什么价值变差、触发后是不再按原建议投注而不是反向下注。
15. 赛前预测必须先做内部「回溯校准」，但回溯只能生成风险审计问题，不能直接改写本场判断。如果最近 5 条已结算记录中同一错因出现 2 次以上，触发「同场证据复核」：只有本场也出现同类脆弱证据，才降低对应方向强度；如果本场首发、战术、天气、盘口价格和比分路径支持该方向，可以继续推荐。最终对用户只能展示翻译后的人话版本，不能直接展示内部标签。
16. 赛后回溯必须联网核对实际比分。没有可靠赛果来源时，写「暂无可靠赛果资料」，不能自行结算。
17. 只要检索到天气，就必须输出「天气对进球/体能影响」；不能只写温度、湿度、风速。
18. 只要检索到确认首发，就必须输出「首发阵容锚点」；不能只写“阵容完整/伤停不明”。
19. 图 A 的主视觉要更像赛前战报海报：毛笔字/冲击字 + 英文小字标签 + 队服/国旗色对抗；不要普通信息卡模板感。
20. 图 A 必须采用「球星/代表球员左右对峙 + 国家视觉符号左右对峙」双主体结构。球队名、盘口和比分是信息层；球员与符号是第一视觉层。
21. 图 A 的国家视觉符号必须可被球迷瞬间识别：优先国家队历史符号、国旗核心图形、绰号意象、队服纹理和城市/球场气质；禁止伪造 FIFA 或官方队徽。
22. 图 B 禁止重新发明 HTML 字体体系。生成前必须先读取一个已验证母版（优先最近正确的 `worldcup-*-analysis.html`，如 `worldcup-qatar-switzerland-v24-analysis.html`；没有时读取 `huxiang-card` 模板），继承其中 `--sans: 'DM Sans', 'KingHwa_OldSong', ...` 与 footer 结构；输出前必须检查 HTML 内存在 `KingHwa_OldSong`，并用截图实际查看 PNG。
23. 必须输出「投注执行建议」，但默认只能使用单位制 `U`，不能主动写人民币/美元具体金额。除非用户明确给出单位金额，否则只写 `主推 1U`、`波胆 0.05U` 这类比例；同时必须解释 `U=自定娱乐单位，不是固定金额`。
24. 投注执行建议必须把主推盘和波胆拆开：主推盘是让球盘或大小球；波胆是小额可选加成，不是主推。
25. 波胆最多建议 1-2 个比分，且必须满足「单个波胆 0.05U-0.10U、合计不超过主推 0.20U」的上限。若没有明显比分收敛，必须写「本场不建议波胆」。
26. 若主推强度为「谨慎可跟 / 可博」，默认主推单位不得超过 `0.5U-0.75U`；若为「中强推荐」，默认 `1U`；若为「强推荐」，也只能写 `1U-1.25U`，不能鼓励加大投入。
27. 禁止任何追损表达：不能写「黑了加倍」「上一场黑这场补」「连红加仓」。连续黑只能触发纪律提醒，不能自动降单位或改通道；是否降单位必须由本场盘口价格、首发不确定性和模型优势厚度决定。
28. 禁止把「球迷喜欢盘口」理解成默认让球盘。球迷常说的盘口包括让球盘和大小球；每场必须显式比较让球盘价值、大小球价值和不投注价值。
29. 如果最终没有选择大小球，必须在人话里说明原因：例如进球路径分歧、盘口从 2.75 降到 2.5 后价值变差、天气/首发信号不够、让球盘容错更好。不能只因为最近大小球黑过、上一场刚推过小球或上一场小球红/黑，就永久降权或机械延续。
30. 如果大小球证据更强，必须主推大小球，即使让球盘也有方向；图片里最大字必须改为「主推：大/小 X.X」，不能继续把球队受让/让胜作为主视觉。
31. 长图和最终输出必须写清「盘口覆盖 ≠ 波胆推荐」。例如主推 `埃及 +1` 时，`1:1 / 0:1 / 2:1走水` 是让球盘结算路径；波胆只推荐 `1:1` 是因为波胆命中率低，不能把所有路径都拿来下注。
32. 18-Agent 真执行硬门槛：如果工具里存在 sub-agent / multi-agent 能力，必须真实创建或分批创建 18 个 Agent 任务；禁止只用一次总搜索、一次浏览器检索或总控自行合成来冒充 18 个 Agent。若因为平台限制无法创建，必须在最终输出中透明说明。
33. 每个 Agent 必须独立提交 `AgentReceipt`。若某 Agent 的资料缺失，状态写 `partial` 或 `blocked`，并明确写「暂无可靠资料」和缺口对最终强度的影响；不能静默跳过。
34. 总预测官必须先读取全部 `AgentReceipt` 后再拍板；如果关键 Agent（赛程、盘口、首发、天气、回溯校准、投注执行）任一为 `blocked`，综合主推最高只能给「谨慎可跟」，必要时改为「本场不建议投注」。
35. 禁止「上盘贵 -> 自动买下盘」：发现热门上盘价格偏贵时，只能触发上盘价值下降和反向复核；下盘必须单独满足概率差、比分覆盖、非盘口证据和风险解释。若下盘未通过，必须改看大小球或不投注，不能为了反热门而反热门。

提取：

- 球队 B

如果只输入两个队名，也直接运行，不要追问。

### 1.5 回溯账本与赛前校准

本技能维护一个本地 JSONL 账本：

```
~/Documents/worldcup-prediction/ledger.jsonl
```

如果文件不存在：

1. 创建 `~/Documents/worldcup-prediction/`。
2. 如果 `data/retrospective-ledger.seed.jsonl` 存在，将种子记录复制/合并为初始账本。
3. 如果种子也不存在，继续预测，但在内部标记「暂无历史回溯记录」。

赛前预测时必须读取账本最近 20 条记录，但必须先声明并执行「本场独立基准」：

1. 先不看上一场红黑，只用本场资料建立胜负、进球区间、让球盘和大小球的初始判断。
2. 再读取账本，把错因转成审计问题，例如「本场是否也存在强队深盘价格过满？」。
3. 只有本场证据回答为「是」，才允许降低强度或换通道；如果本场证据回答为「否」，不得因为历史错因改变结论。

账本最近 20 条记录重点检查：

- 同球队历史预测
- 同阶段（小组赛 / 淘汰赛）
- 同主推盘口类型（让球盘 / 大小球）
- 同错因标签
- 最近 5 条记录里是否有连续错因，但只能作为审计触发器，不能作为自动下注结论

#### 错因标签体系

赛后回溯和赛前校准都使用以下标签：

| 标签 | 含义 | 赛前修正规则 |
|------|------|--------------|
| `small_ball_overuse` | 小球惯性过强，低估早进球/替补冲击 | 不能仅因小球低赔或首战谨慎就推小球 |
| `big_ball_overreach` | 大球想象过多，实际节奏被拖慢 | 推大球必须有双方进球路径或高节奏证据 |
| `home_conversion_overrate` | 高估主场优势和进攻转化 | 主队让球盘必须有真实终结和阵容完整证据 |
| `set_piece_risk_underweight` | 低估定位球/高点改变比分 | 弱方定位球必须进入比分路径 |
| `favorite_handicap_overtrust` | 看对强队但让球盘价格已坏 | 强队让球必须证明价格仍有价值 |
| `score_too_conservative` | 比分预测过保守，低估比赛打开 | 不能机械给 1:0 / 小比分 |
| `high_total_underweight` | 低估比赛打开、强队连续进球或弱队崩盘追分 | 必须做 3+ 球触发器检查；条件成立时高比分必须进入比分池和大小球仲裁 |
| `market_summary_bias` | 顺着盘口解释，没有独立判断 | 必须先写无盘口模型再选盘 |
| `injury_impact_underweight` | 低估核心伤停对结构的影响 | 核心伤停必须影响主推强度 |
| `draw_path_underweight` | 低估弱方拖平路径 | 让半盘必须写清打平即输风险 |

#### 同场证据复核

如果最近 5 条已结算记录里，同一错因标签出现 2 次或以上，先进入复核，不直接熔断：

- 必须先问：本场是否也存在同类风险证据？
- 如果本场不存在同类风险，不降级、不换通道。
- 如果本场存在同类风险，但另有至少 3 条本场强证据支持原方向，可以继续推荐，但强度不得虚高。
- 如果本场存在同类风险且支持证据不足，才降低强度、换通道或不投注。

示例：

- 最近连续错在 `small_ball_overuse`，本场仍可推小球，但必须由本场节奏、首发、天气、盘口关键数和比分路径共同支持；不能只因“小组赛首战谨慎”就推小球。
- 最近连续错在 `favorite_handicap_overtrust`，本场强队仍可让胜，但必须证明盘口门槛没有被名气吃干净；不能因为上一场强队没穿盘就自动反强队。
- 最近连续错在 `home_conversion_overrate`，本场主队让球盘必须证明终结效率和阵容完整；不能只因主场气氛推荐主队。
- 最近连续错在 `set_piece_risk_underweight`，本场必须把弱方定位球写进比分方案。
- 最近连续错在 `high_total_underweight` 或 `score_too_conservative`，本场必须单独回答「有没有 3+ 球触发器」。如果强队有边路/高点/定位球/替补火力，或弱队落后后容易压出崩盘，不能把比分池全部压在 2 球以内。

如果用户输入 `-ledger` 或「看看回溯」「预测账本」「错因统计」，只输出：

```
最近记录：{N} 场
赛果倾向命中：{x}/{N}
主推盘口命中：{x}/{N}
主比分命中：{x}/{N}
比分池命中：{x}/{N}
高频错因：{tag1, tag2, tag3}
下一场校准提醒：{3 条以内}
```

### 2. 赛程确认

检索并确认：

- 是否为 2026 世界杯正式比赛
- 阶段：小组赛 / 32 强 / 16 强 / 8 强 / 半决赛 / 三四名 / 决赛
- 当地开球时间 + 北京时间
- 城市、球场、是否室内或可关闭顶棚
- 两队所在小组/赛程背景
- 淘汰赛是否需要额外判断晋级方

如果没有查到明确对阵，必须写：

「目前未检索到这两队在 2026 世界杯中的明确对阵信息。」

然后继续做「假设性赛前预测」，但必须标注为假设。

### 3. 全维度研究（18-Agent 真执行）

参考 `huxiang-travel` 的 Desk-Based Assessment 思路：不是只查新闻，而是在赛前穷尽可用证据。

#### 18-Agent 执行硬规则

1. 若当前环境支持 sub-agent / Research tool / multi-agent，必须真实派出以下 18 个 Agent。并发不足时分批执行，例如 `1-6`、`7-12`、`13-18`，直到 18 个都返回。
2. 禁止用「8-12 次搜索」冒充 18 个 Agent。搜索只是 Agent 的资料手段，不等于 Agent 已工作。
3. 若当前环境不支持真实 sub-agent，才允许用定向 Web 搜索模拟；但必须在最终输出中写：`18-Agent 状态：当前环境不支持真实并行，已用定向检索模拟，关键缺口为 {缺口}`。
4. 每个 Agent 必须产出一个 `AgentReceipt`，总预测官必须读取全部回执后再拍板。
5. 关键 Agent 不得缺席：Agent 1、3、8、9、10、12、15、16、18 至少必须达到 `completed` 或 `partial`，否则不能给「中强推荐 / 强推荐」。

`AgentReceipt` 标准格式：

```json
{
  "agent_no": 1,
  "agent_name": "赛程确认 Agent",
  "status": "completed | partial | blocked",
  "searched_at_bjt": "2026-06-16T21:30:00+08:00",
  "sources": [
    {"name": "FIFA Match Centre", "url": "https://...", "used_for": "赛程/场地"}
  ],
  "key_findings": ["一句关键发现"],
  "missing_info": ["暂无可靠资料项；没有则为空数组"],
  "impact_on_pick": "增强 / 降低 / 不明显 + 一句话",
  "confidence": "高 / 中 / 低"
}
```

内部可保存回执到：

```
~/Documents/worldcup-prediction/agent-runs/{YYYYMMDD-HHMM}-{match-slug}.json
```

如果保存失败，不中断预测，但最终第 5 部分必须写：`18-Agent 回执保存失败：{原因}`。

最终输出不展示 18 个 Agent 的长篇过程，只展示一行压缩回执：

```
18-Agent 完成回执：18/18 completed，关键缺口：{无 / 首发未确认 / 盘口来源分歧 / 天气缺小时数据}
```

普通球迷不需要看内部长篇，但必须能看到我们不是拍脑袋。

#### Agent 1 — 赛程确认

确认赛程、城市、球场、阶段、出线背景。

#### Agent 2 — 近况状态

查两队最近 5-10 场正式比赛和热身赛：

- 战绩
- 进球/失球
- 攻防走势
- 连胜/连败/状态波动

#### Agent 3 — 阵容伤停与首发

查：

- 最新大名单
- 确认首发
- 预计首发
- 伤病
- 停赛
- 核心球员身体状态
- 轮换风险

首发规则：

- 距开球 90 分钟以内，必须优先检索 FIFA Match Centre、球队官方、权威 live blog、FotMob、Sofascore、FOX/ESPN/Guardian 等确认首发。
- 如果确认首发已出，必须输出 3-6 名最能影响盘口的关键首发，并说明它们如何影响：进攻、控球、定位球、防守高度、反击、体能。
- 如果确认首发未出，必须写「预测首发，以赛前约 1 小时官方名单为准」，并说明不确定性是否降低建议强度。
- 首发不能只是名单堆砌，必须转成一句比赛判断，例如「A 队双前锋首发，意味着大球路径增强」或「B 队主力中卫缺阵，受让保护变差」。

#### Agent 4 — 战术风格

分析：

- 常用阵型
- 进攻方式
- 防守方式
- 反击效率
- 定位球能力
- 中场控制力
- 哪一方更克制对手
- 强队破密集防守工具：边路传中、高点、二点球、远射、定位球、肋部直塞、替补冲击
- 弱队落后后是否会压出、体能是否会断档、防线是否容易连续丢球
- 比赛是否有从 0-2 球区间被推到 3+ 球区间的战术触发点

#### Agent 5 — 关键球员

分析双方核心球员：

- 近期进球/助攻
- 创造机会
- 关键传球
- 防守贡献
- 抗压能力
- 是否能决定低比分比赛
- 是否能连续制造高质量机会，把比赛从 2 球区间推到 3+ 球
- 替补席是否有能在 60 分钟后继续冲击进球的爆点

#### Agent 6 — 历史交锋

整理历史交锋，只作为辅助。禁止机械迷信历史。

#### Agent 7 — 数据模型

综合：

- xG / xGA
- 射门质量
- 防守效率
- 进攻效率
- 门将状态
- 定位球数据
- 低比分/大比分概率
- 3+ 球概率和高比分触发器：早进球、双方进球路径、射门量、禁区触球、定位球、门将状态、弱队体能崩盘
- 强队连续进球能力：领先后是否继续进攻、替补火力、净胜球动机、弱队追分后空间

没有可靠 xG 数据就写「暂无可靠资料」，不要用体感数字替代。

#### Agent 8 — 无盘口独立模型

在读取盘口结论前，先只根据比赛事实给出独立判断：

- 90 分钟倾向：主胜 / 平 / 客胜
- 进球区间：0-1 / 2 / 3+
- 更可能的比赛脚本：压制、拉锯、反击、定位球、混战
- 三个比分路径
- 比赛打开检查：是否存在 3+ 球或 4+ 球路径；如果存在，必须把一个高比分路径列入主比分、次选或冷门比分
- 如果三个比分全部在 0-2 球区间，必须说明为什么早进球、替补冲击、弱队压出、定位球和防线崩盘都不足以推到 3 球以上
- 哪个方向最像「真实比赛」而不是「盘口摘要」

这一步必须在最终判断中保留为「独立模型倾向」。

#### Agent 9 — 盘口市场

检索主流盘口：

- 亚洲让球盘
- 大小球盘口
- 胜平负赔率
- 初盘
- 即时盘
- 盘口升降
- 热门方向
- 市场是否过热

每条盘口必须带来源与检索时间。

#### Agent 10 — 庄家意图 / 反诱导核查

专门审盘口，不审情绪：

- 盘口是否偏浅：强方基本面很强但只浅让，是否在防强方不稳
- 盘口是否偏深：强行抬高门槛，是否在消耗热门资金
- 水位是否反常：热门方向持续降水或升水是否与消息面一致
- 大小球是否诱大/诱小：公众直觉与盘口方向是否相反
- 赔率分歧：主流公司是否一致，还是离散很大
- 热门队是否被过度包装：媒体叙事是否强于真实数据
- 是否触发「不建议投注」：看对方向但价格已坏

注意：不要说庄家一定知道结果。只能判断「盘口价格是否诱导市场行为」。

#### Agent 11 — 非正路 / 爆冷可行性

这一步专门反驳正路结论，寻找能让球迷信服的逆向证据。

必须检查：

- 热门方是否被名气、历史、球星或东道主叙事高估
- 弱方是否有明确可执行武器：定位球、反击、门将、身体、边路速度
- 强方是否存在低转化率、慢热、首战压力、后防搭配问题
- 盘口是否「浅让强队」或「深盘压强队」，导致正路价值被吃掉
- 大小球是否和比赛真实节奏错位
- 低得分足球里，红牌、点球、门将失误、定位球是否足以改变方向
- 反向大球是否成立：如果市场或模型过度压小，是否存在早进球、双方都有进球路径、弱队追分空间、强队替补火力导致比赛打开

输出一个结论：

```
逆路可行性：高 / 中 / 低
逆路方向：{受让方 / 平局 / 大球 / 小球 / 不成立}
最能说服球迷的一条逆路理由：{一句话}
```

#### Agent 12 — 天气场地与进球影响

分析：

- 天气
- 温度
- 湿度
- 风速
- 降雨概率
- 草皮情况
- 是否室内球场
- 海拔
- 时差
- 旅途距离
- 休息天数
- 是否连续作战
- 是否对某一方体能更不利

必须把天气数据翻译成比赛影响：

- 对进球数：更偏大球 / 小球 / 无明显影响
- 对节奏：利于高压逼抢 / 利于慢节奏低位 / 利于长传冲吊
- 对球员状态：是否增加体能消耗、抽筋风险、门将脱手、射门脚感、传中质量
- 对盘口：是否支持让球方穿盘、受让方拖住，或改走大小球

禁止只写「晴 27°C、湿度 41%、风 22 km/h」而不解释影响。

#### Agent 13 — 冷门风险

寻找反向证据：

- 强队隐患
- 弱队优势
- 心理压力
- 裁判尺度
- 点球 / 红牌风险
- 定位球爆点
- 盘口过热风险
- 临场首发风险

#### Agent 14 — 配色、视觉符号与海报字体

为两张图收集：

- 两队主客场球衣主色
- 国旗色
- 球场/城市气质
- 天气与时间给画面带来的色彩
- 哪种色彩对抗最像这场比赛
- 两队最适合入画的核心球星 / 代表性球员：优先当家球星、队长、门面球员、能承载本场判断的关键球员；如果官方肖像不适合或不确定，就使用相同气质的球员半身剪影、背影、号码姿态或动作轮廓
- 两队各自最强国家队视觉符号：例如荷兰狮子、日本红日、德国鹰、法国高卢鸡、阿根廷五月太阳、英格兰三狮意象、巴西桑巴黄绿动势等；符号必须来源于国家队/国旗/绰号/队服文化，禁止胡编官方 logo
- 适合本场的视觉关键词：硬朗 / 湿热 / 雨战 / 高原 / 夜场 / 德比 / 对抗 / 速度

#### Agent 18 — 投注执行与波胆纪律

把最终主推转换成球迷能执行、但不刺激下注的单位化建议。

必须判断：

- 本场是否存在「最优投注建议」：有 / 没有
- 最优通道：让球盘 / 大小球 / 不投注
- 通道仲裁：让球盘、大小球、不投注三者各自的价值分、风险分、放弃理由
- 三通道仲裁必须基于本场独立证据：当前盘口价格、关键数、首发、天气、战术、赛程、市场热度和比分路径；不能把上一场同类通道红/黑作为价值分来源
- 上下盘概率隔离：上盘价格偏贵只能降低上盘评分，不能自动提高下盘评分；下盘评分必须来自下盘自己的模型概率是否高于赔率隐含概率、是否被2个以上比分路径覆盖、是否有首发/战术/天气证据支持、风险是否能解释清楚
- 必须输出上下盘双边校验：上盘隐含概率 vs 模型概率、下盘隐含概率 vs 模型概率；若下盘模型概率没有超过隐含概率至少约2个百分点，禁止把下盘列为最终主推
- 大小球仲裁必须把「大球」和「波胆」分开计算：不能因为单个 `3:1 / 4:0` 波胆难中，就否定 `大 2.5 / 大 3.0` 这种多个比分路径叠加的盘口
- 如果比分池里有 2 个以上高权重路径支持 3+ 球，且大小球盘口仍在 2.5 / 2.75 / 3.0 附近，必须把大球列为强候选；只有价格已坏或节奏证据反向时才能放弃
- 可执行建议优先：先找让球盘和大小球里最接近可投的一项；只有触发至少 2 条不投注熔断条件，才能把「不投注」列为最终通道
- 若最近 2 场主推均为不投注，必须启动「连续不投注复核」，并默认给出低单位可执行方向，除非熔断条件成立
- 通道偏置复核：读取最近 5 场主推类型，如果让球盘或大小球出现 4 场及以上，必须说明这次是否存在惯性偏置；单独上一场不能构成偏置证据
- 主推单位：按强度映射为 `0U / 0.5U / 0.75U / 1U / 1.25U`
- 是否建议波胆：建议 / 不建议
- 波胆比分：最多 1-2 个，只能来自 3 个预测比分里最集中的比分路径
- 波胆单位：单个 `0.05U-0.10U`，合计不超过主推单位的 20%
- 放弃波胆条件：比分路径分散、大小球/让球价值更清晰、波胆赔率不足、临场首发或天气增加随机性
- 反惯性确认：最终主推不是因为上一场强队穿/不穿、上一场小球红/黑、上一场主推类型相同而得出

强度到单位的默认映射：

| 主推强度 | 主推单位建议 |
|----------|--------------|
| 强推荐 | `1U-1.25U` |
| 中强推荐 | `1U` |
| 谨慎可跟 | `0.5U-0.75U` |
| 可博 | `0.25U-0.5U` |
| 不建议 | `0U` |

波胆规则：

- `U` 解释：`1U` 是用户自己设定的主推基准娱乐单位；`0.05U` 是 `1U` 的 5%。例如用户自己把 1U 设为某个固定金额，0.05U 就是这个金额的 5%；模型不替用户设定具体金额。
- 若主推单位为 `1U`，单个波胆最多 `0.05U-0.10U`，两个波胆合计最多 `0.20U`。
- 若主推单位为 `0.5U`，单个波胆最多 `0.025U-0.05U`，两个波胆合计最多 `0.10U`。
- 若本场主推是「不投注」，不得再推荐波胆。
- 若三个比分分别代表完全不同比赛脚本，不得为了覆盖而推荐多个波胆。
- 波胆命中率天然低，只能作为小额观点表达，不能承担主要收益预期。
- 主推盘口的结算路径不等于波胆投注池。若主推让球盘天然覆盖多个比分，仍然只允许从最高收敛比分里选 0-2 个波胆；其他比分只用于解释盘口风险和结算，不写成下注。

输出给最终模板时必须写成人话，例如：

```
投注执行建议：主推让球盘 1U；U=自定娱乐单位，不是固定金额；波胆只小额可选，2:0 可 0.05U，1:0 可 0.05U；不可追加第三个比分。
```

通道仲裁输出给内部判断时使用：

```
让球盘价值：高 / 中 / 低；核心理由；放弃条件
大小球价值：高 / 中 / 低；核心理由；放弃条件
不投注价值：高 / 中 / 低；核心理由
最接近可执行方向：让球盘 / 大小球；方向；为什么比另一条更好
上下盘概率隔离：上盘隐含概率{X}% / 模型概率{Y}%；下盘隐含概率{A}% / 模型概率{B}%；是否存在反买误差{是/否}
不投注熔断：触发 {0/1/2/3...} 条；若少于 2 条，禁止最终不投注
最终通道：让球盘 / 大小球 / 不投注
为什么不是另一个通道：一句话
```

### 4. 交叉判断

总预测官只保留关键结论。判断顺序固定，禁止跳过：

1. 先给无盘口独立模型：胜负倾向、进球区间、比分路径。
2. 加入首发阵容：确认首发 / 预测首发如何改变进攻、定位球、防守和盘口强度。
3. 加入天气场地：天气如何影响进球数、节奏、球员体能和大小球。
4. 做比赛打开检查：是否存在 3+ 球、4+ 球或强队连续进球路径；如果存在，必须让高比分和大球进入候选，不得被默认小比分锚点压掉。
5. 再看盘口：市场和独立模型是否一致。
6. 判断盘口：合理 / 偏浅 / 偏深 / 过热 / 疑似诱盘 / 价格已坏。
7. 强制做非正路推演：如果走逆路，最合理路径是什么；逆路可以是受让、平局，也可以是大球或小球。
8. 应用内部回溯校准，但只能做审计复核：不得推翻 1-7 步形成的本场独立基准，除非本场资料本身也证实同类风险。
9. 选择最终盘口：顺盘 / 逆盘 / 大小球 / 不碰。
10. 如果最终顺盘，必须通过「正路说服力门槛」。
11. 最后做投注执行判断：是否存在最优通道、主推单位多少、波胆是否值得小额附加；没有价值就明确写不投注或不建议波胆。

#### 正路说服力门槛

当最终主推方向与市场热门方向一致时，必须同时满足：

1. 至少 3 条非盘口证据同向：阵容、状态、战术、天气、赛程、数据模型。
2. 盘口价格没有明显被吃干净：不能只是「会赢」，还要「盘仍有价值」。
3. 反向证据已经被解释：弱方武器、爆冷路径、诱盘风险都要能被压住。

如果不满足，不能硬推正路。必须改为：

- 逆路方向
- 大小球方向
- 或「本场不建议投注」

#### 反惯性门槛

任何最终主推都必须通过反惯性检查：

1. 不能因为上一场强队没穿盘，就默认本场强队不穿盘。
2. 不能因为上一场强队穿盘，就默认本场强队继续穿盘。
3. 不能因为上一场刚推小球，就机械继续小球或机械避开小球。
4. 不能因为最近某类盘口红/黑，就把它当成下一场概率证据。
5. 只有本场首发、战术、天气、赛程、盘口关键数、市场热度和比分路径共同支持时，才允许推荐对应通道。

输出内部判断：

```
反惯性检查：{通过/不通过}
本场主推是否来自本场证据：{是/否 + 一句话}
是否受到上一场红黑影响：否
```

#### 比赛打开 / 大球开放门槛

每场必须做一次「比赛是否会打开」检查，不能默认压低比分。

出现以下信号时，必须提高 3+ 球、高比分和大球权重：

- 强队有多种破密集防守方式：边路传中、高点、二点球、远射、定位球、肋部直塞、替补爆点。
- 弱队低位防守但防空、门将、边路、后腰保护或体能存在明显短板。
- 弱队一旦先丢球必须压出，反而给强队二次进球和三球以上空间。
- 双方都有清晰进球路径：强队压制进球 + 弱队反击/定位球偷一个。
- 天气、草皮、场馆条件不抑制射门和传中，甚至利于技术队持续进攻。
- 市场总进球停在 2.5 / 2.75 / 3.0，但独立模型给出多个 3+ 球路径，且大球价格没有明显坏掉。
- 强队有小组净胜球、首战立威、替补竞争或明星进球叙事，领先后仍可能继续进攻。

满足任意 3 条：

- 大球必须进入「大小球候选」，不能只写小球或不碰大小球。
- 三个比分方案里至少一个必须是 3+ 球比分，例如 `2:1 / 3:0 / 3:1 / 4:0`。
- 如果最终仍不主推大球，必须写清放弃理由：价格已坏、首发不支持、天气压节奏、强队领先后收、弱队无进球路径等。

满足任意 4 条且盘口价格未坏：

- `大 2.5 / 大 2.75 / 大 3.0` 可以成为主推，即使公众也偏大球；热门不等于错误，关键是价格是否仍有价值。

内部输出：

```
比赛打开检查：{通过/不通过}
3+球触发器：{列出 2-4 条本场证据}
高比分路径：{例如 2:1 / 3:0 / 3:1 / 4:0}
大球是否进入主推候选：{是/否 + 理由}
若不选大球：{具体放弃原因，不能写“最近小球更稳”}
```

#### 逆路触发器

出现以下信号时，必须认真考虑逆路，不能一句带过：

- 名气强队只浅让，且基本面并没有压倒性优势
- 热门方媒体叙事很强，但近期真实状态一般
- 弱方有稳定定位球/反击/门将/身体优势
- 小组赛首战、淘汰赛首回合式谨慎节奏
- 天气、草皮、海拔、旅途削弱技术优势
- 大小球盘口和公众直觉明显相反
- 盘口朝热门方向降水，但让球不升盘

### 5. 盘口投注建议规则

必须分别判断两条线，最后只给一个综合主推。主推可以是顺盘，也可以是逆盘，也可以是大小球，也可以是不碰。最终主推不是从让球盘里默认挑一个，而是从「让球盘 / 大小球 / 不投注」三通道里选价值最高的一项。

三通道评分只允许使用本场证据。上一场或最近几场的赛果红黑、强队是否穿盘、大小球是否命中，只能提醒检查问题，不能进入本场通道价值分。

但执行优先级必须调整为：

1. 先从让球盘和大小球里找「最优可执行方向」。
2. 再判断这个方向是否通过最低可执行阈值。
3. 只有未通过阈值，或触发不投注熔断条件，才允许最终写不投注。

原因：庄家定价不是为了给用户一个“不投注”的结论，而是为了在强弱、伤停、天气、公众热度和水位利润之间促成成交。总控不能把“庄家也考虑到了”简单翻译成“没有投注价值”；必须在盘口里找相对最优的那条路。没有绝对便宜，仍可能存在相对更好的盘口。

#### 上下盘价格隔离

上盘贵、热门热、低赔难买，只能说明「上盘可能没价值」，不能自动推出「下盘有价值」。

每次让球盘仲裁必须分别回答：

1. 上盘当前价格对应的隐含概率是多少？
2. 本场独立模型给上盘赢盘概率是多少？
3. 下盘当前价格对应的隐含概率是多少？
4. 本场独立模型给下盘赢盘概率是多少？
5. 下盘是否有至少 2 个高权重比分路径覆盖？
6. 下盘是否有首发、战术、天气、赛程、市场补偿中至少 2 项非盘口证据支持？

只有当下盘自己的模型概率高于隐含概率，并且比分路径与非盘口证据同时成立时，才允许因为上盘偏贵而转向下盘。

如果结论是：

```
上盘贵，但下盘概率也不够
```

必须改为：

- 看大小球；
- 降为不投注；
- 或明确写「让球盘不碰」。

禁止写成：

```
上盘价格偏贵，所以主推下盘
```

必须写成：

```
上盘价格偏贵；下盘经独立概率校验也有价值，所以主推下盘
```

#### 主推通道仲裁

每场必须先做通道仲裁，再写综合主推：

| 通道 | 什么时候优先 | 什么时候放弃 |
|------|--------------|--------------|
| 让球盘 | 胜负/不败路径比进球数更清楚，盘口保护合理，冷热没有吃掉价值 | 让球过深、热门过热、强队小胜路径明显、首发不确定影响进攻上限 |
| 大小球 | 比赛节奏、天气、首发、战术和盘口变动共同指向总进球，且比分路径集中 | 双方进球路径分歧大、红牌/点球随机性高、盘口从关键数降档或升档后容错变差 |
| 不投注 | 盘口缺失、盘口价格极坏、首发严重不确定、让球和大小球都没有通过最低可执行阈值 | 只要让球盘或大小球有一个方向通过最低可执行阈值，就必须放弃不投注，给低单位主推 |

#### 最低可执行阈值

让球盘或大小球满足以下任意 3 条，即可视为通过最低可执行阈值，必须给出主推方向，不能写不投注：

- 无盘口独立模型与盘口方向同向，且不是单纯顺热门叙事。
- 盘口门槛可以被 2 个以上高权重比分覆盖。
- 对大小球：盘口方向可以被多个比分路径覆盖。大球覆盖的是一组高比分路径，不是单个波胆；小球覆盖的是一组低比分路径。
- 天气、首发、战术至少有 2 项支持该通道。
- 市场热度没有明显把价格吃干净，或热门方向虽然热但盘口没有继续加深。
- 反向风险清楚但不能推翻主路径。
- 该通道比另外一个投注通道更容易向球迷解释清楚赢盘/输盘逻辑。

通过最低阈值但优势不厚时，建议强度只能写「谨慎可跟」或「可博」，单位为 `0.25U-0.75U`。不能为了给建议而升成中强推荐。

#### 不投注熔断条件

只有出现以下任意 2 条，才允许最终主推「不投注 0U」：

- 可靠盘口缺失，或不同来源盘口差异大到无法确认主线。
- 官方首发未发布且关键球员状态会直接改变让球盘和大小球判断。
- 让球盘与大小球的模型概率都在盈亏平衡线附近，且最大优势不足约 2 个百分点。
- 三个比分路径分散到互相冲突：一条支持让球、一条支持大球、一条支持小球/受让，无法形成主通道。
- 盘口已经跨过关键数导致容错明显变差，例如从 -1.25 升到 -1.5、从 2.75 降到 2.5 后失去走水保护。
- 市场热度与盘口变化同时显示诱买风险，且没有新的首发/战术证据压住风险。

如果只满足 1 条不投注理由，不允许不投注；必须从让球盘或大小球里选一个最优方向，低单位执行。

#### 投注通道定义

输出前必须先区分三件事：

| 通道 | 含义 | 示例 | 是否等同投注主推 |
|------|------|------|------------------|
| 90 分钟胜负倾向 | 对比赛结果的赛果判断 | 加拿大胜、平局、波黑胜 | 否 |
| 让球盘 | 对亚洲让球盘的下注方向 | 加拿大 -0.5、波黑 +0.5 | 可能 |
| 大小球 | 对总进球数的下注方向 | 大 2.5、小 2.25 | 可能 |

如果主推是大小球，胜负倾向只能作为辅助信息，不能写成投注建议。
如果主推是让球盘，必须明确写出让球方向和赢盘含义，例如：`加拿大 -0.5（加拿大 90 分钟赢球才赢盘）`。
如果看不准让球盘但看得准节奏，就主推大小球；如果看不准大小球但让球有价值，就主推让球盘；都没有价值就写「本场不建议投注」。

| 盘口 | 中文说法 | 赢盘规则 | 走水/半赢半输 |
|------|----------|----------|----------------|

#### 盘口失效条件定义

「盘口失效条件」是风控边界，不是第二个投注建议。

必须用球迷能理解的话说明：

- 触发前：当前主推盘口是什么。
- 触发后：盘口保护变少、让球压力变大、赔率价值变差，导致原主推推荐作废。
- 触发含义：不再按原建议执行，不代表自动反向下注。
- 下一步：需要重新检索最新盘口、首发和临场信息后再判断；不能在同一输出里临时改推另一个方向。

示例：

```
盘口失效条件：如果日本 +0.5 退到 +0.25 或平手盘，原「日本 +0.5」推荐作废；这不是改买荷兰，而是需要重新判断。
盘口失效条件：如果库拉索 +3.5 降到 +3，受让保护减少，原「库拉索 +3.5」推荐作废；这不是改买德国穿盘。
盘口失效条件：如果小 2.5 降到小 2.25，容错变低，原「小 2.5」推荐作废；这不是自动改买大球。
```

## Known Pitfalls

- 不要只给胜平负，必须给盘口。
- 不要看到强队就自动让胜；先判断盘口是否已经把优势吃干净。
- 不要因为上一场强队没穿盘，就自动反本场强队；也不要因为上一场强队穿盘，就自动追本场强队。每场必须重新算盘口门槛。
- 不要把让球盘当默认主推。每场都要让「让球盘、大小球、不投注」竞争，谁的价格和比赛逻辑最一致才选谁。
- 不要因为最近大小球黑过，就长期不推大小球；错因只能降低盲目小球/大球，不能封杀大小球通道。
- 不要因为上一场刚推小球，就继续机械推小球或机械避开小球。大小球只能由本场节奏、首发、天气、盘口关键数和比分路径决定。
- 不要把「波胆低命中率」误读成「大球不能推」。大球不是猜一个比分，而是判断多个 3+ 球路径是否合力超过盘口价格。
- 不要把弱队密集防守自动翻译成小球。要先看强队有没有高球、边路、二点、定位球、远射和替补冲击；这些成立时，密集防守也可能被打穿。
- 不要把 2:0 / 2:1 当成所有强弱盘的默认答案。若比赛打开检查成立，主比分或次选比分可以大胆进入 3:0 / 3:1 / 4:0 / 3:2 等高分区。
- 不要因为大球也是热门就天然反大球。热门方向只有在价格被吃干净或本场节奏不支持时才降权；如果本场进球触发器强，顺大球也可以是专业判断。
- 如果连续多场主推让球盘，要主动复核是不是通道惯性，并写清大小球为什么没有让球盘好。
- 不要把热门媒体叙事当作盘口价值。
- 不要顺着盘口写总结。必须先做无盘口独立模型，再做盘口对照。
- 不要为了显得专业而永远正路。足球低比分、高波动，逆路要被认真建模。
- 如果最终仍然顺盘，必须写清为什么这次顺盘不是被市场牵着走。
- 不要用「看好某队」承载投注建议。必须拆成「90分钟胜负倾向」和「主推盘口类型」。
- 如果主推大小球，海报最大字必须是「主推：大/小 X.X」，不是球队名。
- 如果主推让球盘，必须写赢盘条件，避免用户把胜负倾向误读成让球推荐。
- 如果主推让半球、平半、半一、一球等亚洲盘，必须把赢盘、输盘、走水、赢半/输半条件说清楚。
- 不要假设用户理解 `-0.25`、`-0.5`、`+0.75`；投注建议必须翻译成人话。
- 不要只写「不追」「升盘不追」「降盘不追」。必须写「盘口失效条件」，并说明这是原推荐作废，不是改买反方向。
- 不要在同一条主推后又临时给第二个方向。盘口失效后只能写「需要重新判断」，不能写成隐藏备选投注。
- 不要因为怀疑诱盘就不敢下结论；疑似诱盘要写清触发证据。
- 第二张分析长图必须保留 `huxiang-card` footer：logo + 虎小象。
- 不要把内部回溯标签展示给球迷。`score_too_conservative` 这类词只能留在内部账本，输出时改写成「赛前证据校准」「风险提醒」。
- 不要只列天气数据。必须写它对节奏、体能、射门、门将和大小球的影响。
- 不要只说「阵容完整」。查得到确认首发时，必须列关键首发并说明如何影响盘口。
- 不要把波胆写成主推。波胆命中率天然低，只能小额可选；主推单位必须明显大于波胆合计。
- 不要让用户误会 `U` 是固定金额或模型指定金额。每次出现 `U`，必须说明这是用户自定娱乐单位。
- 不要把「盘口覆盖某比分」写得像「波胆也要买这个比分」。盘口覆盖是结算逻辑，波胆推荐是单独的小概率投注，两者必须分层。
- 不要为了显得覆盖全面而推荐 3 个以上波胆比分。三个预测比分是比赛路径，不等于三个下注比分。
- 不要输出具体金额，除非用户明确给了预算或单位金额。默认用 `U` 单位，且任何建议都必须限定在娱乐预算内。
- 不要把「最优投注建议」理解成必须下注。没有价格优势、比分分散、盘口已坏时，最优建议就是不投注或只看球。
- 不要生成普通信息卡风格海报。图 A 必须有毛笔字/手写冲击字、英文功能小字和强对抗构图，且主推盘口最大。
- 不要让图 A 只有文字和背景球场。必须同时有两队当家球星/代表性球员、国家视觉符号和队服/国旗色对抗；例如荷兰要想到狮子，日本要想到红日，这类符号由 Agent 14 每场先定。
- 不要接受普通黑体标题冒充毛笔字。图 A 顶部主标题必须有书写感、笔触感或体育战报冲击感；如果不达标，减少文案后重生。
- 不要在图 B 里临时自造字体和排版。必须先继承已验证母版，尤其是 `KingHwa_OldSong`（京華老宋体）字体路径；否则长图会变成普通 dashboard，不符合本技能。
