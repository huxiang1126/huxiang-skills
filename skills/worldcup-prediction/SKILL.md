# worldcup-prediction: 世界杯赛前盘口预测总控

一条命令完成：球队解析 -> 内部回溯账本读取 -> 最新资料检索 -> 首发/天气/盘口核查 -> 无盘口独立建模 -> 多 Agent 交叉判断 -> 盘口价值/诱盘核查 -> 正路/逆路校准 -> 单位化投注执行建议 -> 球迷可读结论 -> 两张朋友圈图 -> 本地预测账本记录。

核心原则：

- 先检索，后结论。
- 查不到就写「暂无可靠资料」。
- 不编盘口、不编伤停、不编天气。
- 必须给出一个唯一主推盘口；如果没有价值，明确写「本场不建议投注」。
- 盘口不是预知赛果，盘口是市场价格和风险管理信号；必须识别过热、偏浅、偏深和疑似诱盘，但不能阴谋论化。
- 盘口不能当答案。必须先给出「不看盘口时的独立模型倾向」，再和盘口对照。
- 不能默认正路。最终可以顺盘，也可以逆盘，也可以不碰；关键是价格和比赛逻辑是否匹配。
- 如果最终顺着市场热门方向，必须额外说明「为什么这次顺盘仍然有价值」。
- 存在「最优投注建议」，但不是每场都有。最优不等于最刺激，也不等于波胆；它必须同时满足方向清楚、价格未坏、风险可解释、资金效率更好。若没有价值，最优建议就是「不投注」。
- 「胜负倾向」和「投注主推」必须分离。胜负倾向只回答 90 分钟赛果；投注主推必须明确是让球盘、大小球，还是不碰。
- 「投注主推」和「波胆」必须分离。让球盘/大小球才是主执行通道；波胆只能是小额附加，不得喧宾夺主。
- 默认使用单位制 `U` 给执行建议，不输出具体金额，除非用户明确提供预算或单位金额。`1U` 表示一场普通主推的基准娱乐投入。
- 波胆投注比例必须明显小于让球盘/大小球主推：单个波胆建议不超过主推投入的 5%-10%，单场全部波胆合计不超过主推投入的 20%。比分分散、盘口更优、赔率不划算时，必须写「不建议波胆」。
- 最多给 1-2 个可选波胆比分；禁止为了覆盖焦虑列 3 个以上波胆下注方向。三个预测比分是分析路径，不等于三个都要下注。
- 禁止追损、翻倍、加仓追回、连续黑后扩大单位。所有投注执行建议必须限定在娱乐预算内。
- 禁止只写「看好 {球队}」作为海报或最终投注信息。必须写成「90分钟倾向：{球队}胜 / 平」和「主推投注：{盘口类型} {方向}」。
- 如果主推是让球盘，必须把盘口符号翻译成球迷能直接理解的规则：赢盘、输盘、走水、赢半、输半分别在什么比分条件下发生。
- 禁止只写「不追」「不追高」「降盘不追」作为风控说明。必须写成「盘口失效条件」，并说明触发后是原主推推荐作废，不代表自动改买反方向；需要重新检索盘口和首发后再判断。
- 每次赛前预测前必须读取本地回溯账本，检查最近错因；每次赛后复盘必须把结果、盘口命中、比分命中和错因标签写入账本。
- 回溯账本和错因标签是内部模型语言，不能原样展示给普通球迷。最终输出和图片里必须翻译成「赛前证据校准」「临场修正」「风险提醒」这类可信、人话表达。
- 天气不能只列数据。必须说明它对进球数、比赛节奏、体能、逼抢、射门质量、门将处理球或定位球的具体影响。
- 若距开球约 90 分钟内，必须优先抓取官方/权威媒体确认首发；若还没有确认首发，必须标注「预测首发」，并说明首发不确定性对盘口强度的影响。
- 预测海报必须加强视觉冲击：允许毛笔字/手写冲击字、英文功能小字、强对比构图，但仍要清楚写出主推盘口和赢盘条件。
- 预测海报不能只是信息卡。必须出现两队核心球星/代表性球员或可识别的球员剪影，并为两队各配置一个国家队视觉符号，例如荷兰狮子、日本红日、德国鹰、法国高卢鸡、阿根廷太阳纹、英格兰三狮意象等；不确定时使用国旗几何、队服色和城市球场元素，不能自编官方 logo。
- 图 A 顶部主标题必须有毛笔字/手写冲击字气势；若生成结果变成普通黑体、PPT 标题或文字缺字，必须再生成一版更少字、更强主标题的海报。
- 分析思路长图必须继承已验证的 `huxiang-card` 字体和版式母版；中文正文/标题优先走 `KingHwa_OldSong`（京華老宋体）渲染路径，不能临时改成系统 UI 字体或自造 dashboard 风格。

## 硬规则

1. 必须联网检索最新资料。优先官方、权威体育媒体、球队官方、可靠数据/盘口平台。
2. 任何具体数字必须有来源：赛程、比分、近况、伤停、盘口、赔率、天气。
3. 盘口必须写检索时间；超过 24 小时的盘口要标注「可能已变化」。
4. 找不到可靠盘口，写「暂无可靠盘口数据」，不要自己估。
5. 不输出完整 Agent 长篇过程，只输出固定结果。
6. 不使用「梭哈」「稳赚」「必中」「重注」等刺激下注词。
7. 必须提醒：赛前个人分析，仅供娱乐参考，不保证赛果。
8. 图片不能出现 FIFA 官方 logo、投注平台品牌、赌场元素。
9. 禁止把盘口当作结论来源。先做无盘口判断，再做盘口对照。
10. 禁止为了显得稳而永远选热门方、让球方、小球方。足球是低得分高波动项目，必须保留逆路可能。
11. 禁止使用模糊投注词：`看好加拿大`、`看好阿根廷`、`站法国` 不能单独出现，必须标注它是「90分钟胜负倾向」还是「让球盘投注方向」。
12. 海报里最醒目的主信息必须是「主推投注」，不是胜负倾向；如果主推大小球，就不能把球队名做成最大结论。
13. 主推让球盘时，禁止只写 `{球队} -0.5` 或 `{球队} +0.25`。必须同时写「让胜/受让」和赢盘规则。
14. 「盘口失效条件」必须明确回答三件事：哪个盘口变化会让原推荐失效、为什么价值变差、触发后是不再按原建议投注而不是反向下注。
15. 赛前预测必须先做内部「回溯校准」。如果最近 5 条已结算记录中同一错因出现 2 次以上，触发熔断：不能继续推荐同类脆弱方向，除非有 3 条新的强证据。最终对用户只能展示翻译后的人话版本，不能直接展示内部标签。
16. 赛后回溯必须联网核对实际比分。没有可靠赛果来源时，写「暂无可靠赛果资料」，不能自行结算。
17. 只要检索到天气，就必须输出「天气对进球/体能影响」；不能只写温度、湿度、风速。
18. 只要检索到确认首发，就必须输出「首发阵容锚点」；不能只写“阵容完整/伤停不明”。
19. 图 A 的主视觉要更像赛前战报海报：毛笔字/冲击字 + 英文小字标签 + 队服/国旗色对抗；不要普通信息卡模板感。
20. 图 A 必须采用「球星/代表球员左右对峙 + 国家视觉符号左右对峙」双主体结构。球队名、盘口和比分是信息层；球员与符号是第一视觉层。
21. 图 A 的国家视觉符号必须可被球迷瞬间识别：优先国家队历史符号、国旗核心图形、绰号意象、队服纹理和城市/球场气质；禁止伪造 FIFA 或官方队徽。
22. 图 B 禁止重新发明 HTML 字体体系。生成前必须先读取一个已验证母版（优先最近正确的 `worldcup-*-analysis.html`，如 `worldcup-qatar-switzerland-v24-analysis.html`；没有时读取 `huxiang-card` 模板），继承其中 `--sans: 'DM Sans', 'KingHwa_OldSong', ...` 与 footer 结构；输出前必须检查 HTML 内存在 `KingHwa_OldSong`，并用截图实际查看 PNG。
23. 必须输出「投注执行建议」，但默认只能使用单位制 `U`，不能主动写人民币/美元具体金额。除非用户明确给出单位金额，否则只写 `主推 1U`、`波胆 0.05U` 这类比例。
24. 投注执行建议必须把主推盘和波胆拆开：主推盘是让球盘或大小球；波胆是小额可选加成，不是主推。
25. 波胆最多建议 1-2 个比分，且必须满足「单个波胆 0.05U-0.10U、合计不超过主推 0.20U」的上限。若没有明显比分收敛，必须写「本场不建议波胆」。
26. 若主推强度为「谨慎可跟 / 可博」，默认主推单位不得超过 `0.5U-0.75U`；若为「中强推荐」，默认 `1U`；若为「强推荐」，也只能写 `1U-1.25U`，不能鼓励加大投入。
27. 禁止任何追损表达：不能写「黑了加倍」「上一场黑这场补」「连红加仓」。连续黑或盘口不稳时，必须降单位或不投注。

## 执行

### 1. 解析输入

提取：

- 球队 A
- 球队 B
- 用户是否指定阶段、时间、场地
- 是否是淘汰赛
- 是否为 `-review` 赛后回溯模式
- 是否为 `-ledger` 账本摘要模式

如果只输入两个队名，也直接运行，不要追问。

### 1.5 回溯账本与赛前校准

本技能维护一个本地 JSONL 账本：

```
~/Documents/worldcup-prediction/ledger.jsonl
```

如果文件不存在：

1. 创建 `~/Documents/worldcup-prediction/`。
2. 如果 `data/retrospective-ledger.seed.jsonl` 存在，将种子记录复制/合并为初始账本。
3. 如果种子也不存在，继续预测，但在内部标记「暂无历史回溯记录」。

赛前预测时必须读取账本最近 20 条记录，重点检查：

- 同球队历史预测
- 同阶段（小组赛 / 淘汰赛）
- 同主推盘口类型（让球盘 / 大小球）
- 同错因标签
- 最近 5 条记录里是否有连续错因

#### 错因标签体系

赛后回溯和赛前校准都使用以下标签：

| 标签 | 含义 | 赛前修正规则 |
|------|------|--------------|
| `small_ball_overuse` | 小球惯性过强，低估早进球/替补冲击 | 不能仅因小球低赔或首战谨慎就推小球 |
| `big_ball_overreach` | 大球想象过多，实际节奏被拖慢 | 推大球必须有双方进球路径或高节奏证据 |
| `home_conversion_overrate` | 高估主场优势和进攻转化 | 主队让球盘必须有真实终结和阵容完整证据 |
| `set_piece_risk_underweight` | 低估定位球/高点改变比分 | 弱方定位球必须进入比分路径 |
| `favorite_handicap_overtrust` | 看对强队但让球盘价格已坏 | 强队让球必须证明价格仍有价值 |
| `score_too_conservative` | 比分预测过保守，低估比赛打开 | 不能机械给 1:0 / 小比分 |
| `market_summary_bias` | 顺着盘口解释，没有独立判断 | 必须先写无盘口模型再选盘 |
| `injury_impact_underweight` | 低估核心伤停对结构的影响 | 核心伤停必须影响主推强度 |
| `draw_path_underweight` | 低估弱方拖平路径 | 让半盘必须写清打平即输风险 |

#### 连续错因熔断

如果最近 5 条已结算记录里，同一错因标签出现 2 次或以上：

- 该错因对应的方向自动降一级强度。
- 如果本场仍要推荐同类方向，必须写出 3 条新的强证据。
- 如果证据不足，改为另一个盘口类型或「本场不建议投注」。

示例：

- 最近连续错在 `small_ball_overuse`，本场不能只因“小组赛首战谨慎”就推小球。
- 最近连续错在 `home_conversion_overrate`，本场主队 -0.5 必须证明终结效率和阵容完整。
- 最近连续错在 `set_piece_risk_underweight`，本场必须把弱方定位球写进比分方案。

#### 账本摘要输出

如果用户输入 `-ledger` 或「看看回溯」「预测账本」「错因统计」，只输出：

```
最近记录：{N} 场
赛果倾向命中：{x}/{N}
主推盘口命中：{x}/{N}
主比分命中：{x}/{N}
比分池命中：{x}/{N}
高频错因：{tag1, tag2, tag3}
下一场校准提醒：{3 条以内}
```

### 2. 赛程确认

检索并确认：

- 是否为 2026 世界杯正式比赛
- 阶段：小组赛 / 32 强 / 16 强 / 8 强 / 半决赛 / 三四名 / 决赛
- 当地开球时间 + 北京时间
- 城市、球场、是否室内或可关闭顶棚
- 两队所在小组/赛程背景
- 淘汰赛是否需要额外判断晋级方

如果没有查到明确对阵，必须写：

「目前未检索到这两队在 2026 世界杯中的明确对阵信息。」

然后继续做「假设性赛前预测」，但必须标注为假设。

### 3. 全维度研究（多 Agent 并行）

参考 `huxiang-travel` 的 Desk-Based Assessment 思路：不是只查新闻，而是在赛前穷尽可用证据。若当前环境支持 subagent / Research tool，派出以下 Agent 并行；若不支持，就用 8-12 次定向 Web 搜索模拟这些 Agent。

#### Agent 1 — 赛程确认

确认赛程、城市、球场、阶段、出线背景。

#### Agent 2 — 近况状态

查两队最近 5-10 场正式比赛和热身赛：

- 战绩
- 进球/失球
- 攻防走势
- 连胜/连败/状态波动

#### Agent 3 — 阵容伤停与首发

查：

- 最新大名单
- 确认首发
- 预计首发
- 伤病
- 停赛
- 核心球员身体状态
- 轮换风险

首发规则：

- 距开球 90 分钟以内，必须优先检索 FIFA Match Centre、球队官方、权威 live blog、FotMob、Sofascore、FOX/ESPN/Guardian 等确认首发。
- 如果确认首发已出，必须输出 3-6 名最能影响盘口的关键首发，并说明它们如何影响：进攻、控球、定位球、防守高度、反击、体能。
- 如果确认首发未出，必须写「预测首发，以赛前约 1 小时官方名单为准」，并说明不确定性是否降低建议强度。
- 首发不能只是名单堆砌，必须转成一句比赛判断，例如「A 队双前锋首发，意味着大球路径增强」或「B 队主力中卫缺阵，受让保护变差」。

#### Agent 4 — 战术风格

分析：

- 常用阵型
- 进攻方式
- 防守方式
- 反击效率
- 定位球能力
- 中场控制力
- 哪一方更克制对手

#### Agent 5 — 关键球员

分析双方核心球员：

- 近期进球/助攻
- 创造机会
- 关键传球
- 防守贡献
- 抗压能力
- 是否能决定低比分比赛

#### Agent 6 — 历史交锋

整理历史交锋，只作为辅助。禁止机械迷信历史。

#### Agent 7 — 数据模型

综合：

- xG / xGA
- 射门质量
- 防守效率
- 进攻效率
- 门将状态
- 定位球数据
- 低比分/大比分概率

没有可靠 xG 数据就写「暂无可靠资料」，不要用体感数字替代。

#### Agent 8 — 无盘口独立模型

在读取盘口结论前，先只根据比赛事实给出独立判断：

- 90 分钟倾向：主胜 / 平 / 客胜
- 进球区间：0-1 / 2 / 3+
- 更可能的比赛脚本：压制、拉锯、反击、定位球、混战
- 三个比分路径
- 哪个方向最像「真实比赛」而不是「盘口摘要」

这一步必须在最终判断中保留为「独立模型倾向」。

#### Agent 9 — 盘口市场

检索主流盘口：

- 亚洲让球盘
- 大小球盘口
- 胜平负赔率
- 初盘
- 即时盘
- 盘口升降
- 热门方向
- 市场是否过热

每条盘口必须带来源与检索时间。

#### Agent 10 — 庄家意图 / 反诱导核查

专门审盘口，不审情绪：

- 盘口是否偏浅：强方基本面很强但只浅让，是否在防强方不稳
- 盘口是否偏深：强行抬高门槛，是否在消耗热门资金
- 水位是否反常：热门方向持续降水或升水是否与消息面一致
- 大小球是否诱大/诱小：公众直觉与盘口方向是否相反
- 赔率分歧：主流公司是否一致，还是离散很大
- 热门队是否被过度包装：媒体叙事是否强于真实数据
- 是否触发「不建议投注」：看对方向但价格已坏

注意：不要说庄家一定知道结果。只能判断「盘口价格是否诱导市场行为」。

#### Agent 11 — 非正路 / 爆冷可行性

这一步专门反驳正路结论，寻找能让球迷信服的逆向证据。

必须检查：

- 热门方是否被名气、历史、球星或东道主叙事高估
- 弱方是否有明确可执行武器：定位球、反击、门将、身体、边路速度
- 强方是否存在低转化率、慢热、首战压力、后防搭配问题
- 盘口是否「浅让强队」或「深盘压强队」，导致正路价值被吃掉
- 大小球是否和比赛真实节奏错位
- 低得分足球里，红牌、点球、门将失误、定位球是否足以改变方向

输出一个结论：

```
逆路可行性：高 / 中 / 低
逆路方向：{受让方 / 平局 / 大球 / 小球 / 不成立}
最能说服球迷的一条逆路理由：{一句话}
```

#### Agent 12 — 天气场地与进球影响

分析：

- 天气
- 温度
- 湿度
- 风速
- 降雨概率
- 草皮情况
- 是否室内球场
- 海拔
- 时差
- 旅途距离
- 休息天数
- 是否连续作战
- 是否对某一方体能更不利

必须把天气数据翻译成比赛影响：

- 对进球数：更偏大球 / 小球 / 无明显影响
- 对节奏：利于高压逼抢 / 利于慢节奏低位 / 利于长传冲吊
- 对球员状态：是否增加体能消耗、抽筋风险、门将脱手、射门脚感、传中质量
- 对盘口：是否支持让球方穿盘、受让方拖住，或改走大小球

禁止只写「晴 27°C、湿度 41%、风 22 km/h」而不解释影响。

#### Agent 13 — 冷门风险

寻找反向证据：

- 强队隐患
- 弱队优势
- 心理压力
- 裁判尺度
- 点球 / 红牌风险
- 定位球爆点
- 盘口过热风险
- 临场首发风险

#### Agent 14 — 配色、视觉符号与海报字体

为两张图收集：

- 两队主客场球衣主色
- 国旗色
- 球场/城市气质
- 天气与时间给画面带来的色彩
- 哪种色彩对抗最像这场比赛
- 两队最适合入画的核心球星 / 代表性球员：优先当家球星、队长、门面球员、能承载本场判断的关键球员；如果官方肖像不适合或不确定，就使用相同气质的球员半身剪影、背影、号码姿态或动作轮廓
- 两队各自最强国家队视觉符号：例如荷兰狮子、日本红日、德国鹰、法国高卢鸡、阿根廷五月太阳、英格兰三狮意象、巴西桑巴黄绿动势等；符号必须来源于国家队/国旗/绰号/队服文化，禁止胡编官方 logo
- 适合本场的视觉关键词：硬朗 / 湿热 / 雨战 / 高原 / 夜场 / 德比 / 对抗 / 速度
- 中文主标题的字体气质：优先毛笔字 / 手写冲击字；只有当比赛气质明显更冷静时才用压迫感黑体、报纸战报字、金属刻字。每场只选一种主风格，并写入图 A 提示词
- 主标题气势检查：标题必须像体育战报封面，不像 PPT 标题；如果生成结果没有毛笔/手写冲击感，必须用更短文案重生
- 英文功能小字：FINAL PICK / HANDICAP / SCORE / WEATHER / STARTING XI，用来增强设计感，不替代中文判断
- 图 B 长图字体母版：优先继承上一张已验证 `worldcup-*-analysis.html` 的字体栈和版式骨架；中文正文/标题必须保留 `KingHwa_OldSong`（京華老宋体）作为中文渲染路径，不能用系统 UI 字体重写。

输出给图片生成与 `huxiang-card` 卡片使用。

#### Agent 15 — 内部回溯校准

读取本地账本，生成赛前校准提醒：

- 最近 5 场最常见错因
- 是否触发连续错因熔断
- 本场最需要避免的一个模型习惯
- 本场是否需要降低某个盘口方向强度

输出格式：

```
内部回溯校准：{一句话，允许使用错因标签，仅供内部判断}
触发熔断：{无 / 标签名 + 原因}
本场修正：{一句话}
```

注意：这一步的错因标签不能原样进入最终输出、朋友圈文案或图片。

#### Agent 16 — 首发可信度

专门检查首发资料的可靠程度：

- 已确认首发：官方/FIFA/权威 live blog 已发布
- 高可信预测首发：多家权威媒体一致
- 低可信预测首发：来源少或互相冲突
- 暂无可靠首发：没有可靠资料

输出：

```
首发可信度：已确认 / 高 / 中 / 低 / 暂无可靠资料
关键首发锚点：{3-6 名球员 + 对比赛脚本的影响}
对盘口强度影响：增强 / 降低 / 不明显 + 一句话
```

#### Agent 17 — 球迷可信表达

把内部模型语言翻译成球迷愿意相信的证据语言。

必须转换：

- `score_too_conservative` -> 「这场不能只按 1:0 的保守脚本看，双方都有打开比分的触发点」
- `set_piece_risk_underweight` -> 「弱方定位球/高点/二点球必须进入比分路径」
- `market_summary_bias` -> 「先看比赛本身，再看盘口是否给了合适价格」
- `home_conversion_overrate` -> 「主场气氛不等于进球转化，必须看首发终结点」
- `big_ball_overreach` -> 「有进攻想象不等于一定大球，必须看节奏和天气」

最终输出和图片里使用：

```
赛前证据校准：{人话提醒 + 来源/原因}
```

禁止输出：

```
score_too_conservative 触发
set_piece_risk_underweight 触发
历史错因校准
回溯校准
```

#### Agent 18 — 投注执行与波胆纪律

把最终主推转换成球迷能执行、但不刺激下注的单位化建议。

必须判断：

- 本场是否存在「最优投注建议」：有 / 没有
- 最优通道：让球盘 / 大小球 / 不投注
- 主推单位：按强度映射为 `0U / 0.5U / 0.75U / 1U / 1.25U`
- 是否建议波胆：建议 / 不建议
- 波胆比分：最多 1-2 个，只能来自 3 个预测比分里最集中的比分路径
- 波胆单位：单个 `0.05U-0.10U`，合计不超过主推单位的 20%
- 放弃波胆条件：比分路径分散、大小球/让球价值更清晰、波胆赔率不足、临场首发或天气增加随机性

强度到单位的默认映射：

| 主推强度 | 主推单位建议 |
|----------|--------------|
| 强推荐 | `1U-1.25U` |
| 中强推荐 | `1U` |
| 谨慎可跟 | `0.5U-0.75U` |
| 可博 | `0.25U-0.5U` |
| 不建议 | `0U` |

波胆规则：

- 若主推单位为 `1U`，单个波胆最多 `0.05U-0.10U`，两个波胆合计最多 `0.20U`。
- 若主推单位为 `0.5U`，单个波胆最多 `0.025U-0.05U`，两个波胆合计最多 `0.10U`。
- 若本场主推是「不投注」，不得再推荐波胆。
- 若三个比分分别代表完全不同比赛脚本，不得为了覆盖而推荐多个波胆。
- 波胆命中率天然低，只能作为小额观点表达，不能承担主要收益预期。

输出给最终模板时必须写成人话，例如：

```
投注执行建议：主推让球盘 1U；波胆只小额可选，2:0 可 0.05U，1:0 可 0.05U；不可追加第三个比分。
```

### 4. 交叉判断

总预测官只保留关键结论。判断顺序固定，禁止跳过：

1. 先给无盘口独立模型：胜负倾向、进球区间、比分路径。
2. 加入首发阵容：确认首发 / 预测首发如何改变进攻、定位球、防守和盘口强度。
3. 加入天气场地：天气如何影响进球数、节奏、球员体能和大小球。
4. 再看盘口：市场和独立模型是否一致。
5. 判断盘口：合理 / 偏浅 / 偏深 / 过热 / 疑似诱盘 / 价格已坏。
6. 强制做非正路推演：如果走逆路，最合理路径是什么。
7. 应用内部回溯校准，并翻译成「赛前证据校准」的人话表达。
8. 选择最终盘口：顺盘 / 逆盘 / 大小球 / 不碰。
9. 如果最终顺盘，必须通过「正路说服力门槛」。
10. 最后做投注执行判断：是否存在最优通道、主推单位多少、波胆是否值得小额附加；没有价值就明确写不投注或不建议波胆。

#### 正路说服力门槛

当最终主推方向与市场热门方向一致时，必须同时满足：

1. 至少 3 条非盘口证据同向：阵容、状态、战术、天气、赛程、数据模型。
2. 盘口价格没有明显被吃干净：不能只是「会赢」，还要「盘仍有价值」。
3. 反向证据已经被解释：弱方武器、爆冷路径、诱盘风险都要能被压住。

如果不满足，不能硬推正路。必须改为：

- 逆路方向
- 大小球方向
- 或「本场不建议投注」

#### 逆路触发器

出现以下信号时，必须认真考虑逆路，不能一句带过：

- 名气强队只浅让，且基本面并没有压倒性优势
- 热门方媒体叙事很强，但近期真实状态一般
- 弱方有稳定定位球/反击/门将/身体优势
- 小组赛首战、淘汰赛首回合式谨慎节奏
- 天气、草皮、海拔、旅途削弱技术优势
- 大小球盘口和公众直觉明显相反
- 盘口朝热门方向降水，但让球不升盘

### 5. 盘口投注建议规则

必须分别判断两条线，最后只给一个综合主推。主推可以是顺盘，也可以是逆盘，也可以是大小球，也可以是不碰。

#### 投注通道定义

输出前必须先区分三件事：

| 通道 | 含义 | 示例 | 是否等同投注主推 |
|------|------|------|------------------|
| 90 分钟胜负倾向 | 对比赛结果的赛果判断 | 加拿大胜、平局、波黑胜 | 否 |
| 让球盘 | 对亚洲让球盘的下注方向 | 加拿大 -0.5、波黑 +0.5 | 可能 |
| 大小球 | 对总进球数的下注方向 | 大 2.5、小 2.25 | 可能 |

如果主推是大小球，胜负倾向只能作为辅助信息，不能写成投注建议。
如果主推是让球盘，必须明确写出让球方向和赢盘含义，例如：`加拿大 -0.5（加拿大 90 分钟赢球才赢盘）`。
如果看不准让球盘但看得准节奏，就主推大小球；如果看不准大小球但让球有价值，就主推让球盘；都没有价值就写「本场不建议投注」。

#### 亚洲让球盘解释表

凡是推荐让球盘，必须根据盘口写清规则。常见盘口解释如下：

| 盘口 | 中文说法 | 赢盘规则 | 走水/半赢半输 |
|------|----------|----------|----------------|
| `球队 0` | 平手盘 | 该队 90 分钟赢球则赢盘 | 打平走水，输球输盘 |
| `球队 -0.25` | 让平/半 | 该队赢球全赢 | 打平输半，输球全输 |
| `球队 -0.5` | 让半球 | 该队 90 分钟赢球才赢盘 | 打平或输球都输盘，无走水 |
| `球队 -0.75` | 让半/一 | 该队赢 2 球及以上全赢 | 赢 1 球赢半，打平或输球全输 |
| `球队 -1` | 让一球 | 该队赢 2 球及以上全赢 | 赢 1 球走水，打平或输球全输 |
| `球队 +0.25` | 受让平/半 | 该队赢球全赢 | 打平赢半，输球全输 |
| `球队 +0.5` | 受让半球 | 该队 90 分钟不败即赢盘 | 输球才输盘，无走水 |
| `球队 +0.75` | 受让半/一 | 该队赢球或打平全赢 | 输 1 球输半，输 2 球及以上全输 |
| `球队 +1` | 受让一球 | 该队赢球或打平全赢 | 输 1 球走水，输 2 球及以上全输 |

输出时不要只复制表格，要按本场主推盘口改写成一句人话：

```
让球盘中文解释：加拿大 -0.5 = 加拿大让半球。
赢盘条件：加拿大 90 分钟赢球才赢盘；打平或输球都输盘，没有走水。
```

#### 让球盘

输出：

- 方向：例如 `韩国 -0.25`、`捷克 +0.25`、`不碰让球盘`
- 强度：强推荐 / 中强推荐 / 可博 / 谨慎可跟 / 不建议
- 理由：一句话
- 盘口失效条件：例如「如果从 -0.5 升到 -0.75，原主推价值下降，原推荐作废；不是改买对手，需重新判断」

#### 大小球

输出：

- 方向：例如 `小 2.5`、`大 2.25`、`不碰大小球`
- 强度：强推荐 / 中强推荐 / 可博 / 谨慎可跟 / 不建议
- 理由：一句话
- 盘口失效条件：例如「如果小 2.5 降到小 2.25，进球容错变少，原小 2.5 推荐作废；不是自动改买大球」

#### 盘口失效条件定义

「盘口失效条件」是风控边界，不是第二个投注建议。

必须用球迷能理解的话说明：

- 触发前：当前主推盘口是什么。
- 触发后：盘口保护变少、让球压力变大、赔率价值变差，导致原主推推荐作废。
- 触发含义：不再按原建议执行，不代表自动反向下注。
- 下一步：需要重新检索最新盘口、首发和临场信息后再判断；不能在同一输出里临时改推另一个方向。

示例：

```
盘口失效条件：如果日本 +0.5 退到 +0.25 或平手盘，原「日本 +0.5」推荐作废；这不是改买荷兰，而是需要重新判断。
盘口失效条件：如果库拉索 +3.5 降到 +3，受让保护减少，原「库拉索 +3.5」推荐作废；这不是改买德国穿盘。
盘口失效条件：如果小 2.5 降到小 2.25，容错变低，原「小 2.5」推荐作废；这不是自动改买大球。
```

#### 综合主推

只能从让球盘或大小球中选一个。

在写综合主推前，必须先标注：

```
盘口选择类型：顺盘 / 逆盘 / 大小球 / 不碰
为什么不是盘口摘要：{一句话说明独立模型和盘口之间的差异}
```

如果出现以下任一情况，综合主推改为：

「本场不建议投注」

触发条件：

- 盘口数据缺失
- 盘口疑似诱盘且无法解释
- 首发/伤停信息严重不足
- 极端天气或大轮换导致变数过大
- 看对方向但盘口价格已坏，没有价值差

### 6. 输出模板

最终只输出以下五部分。

#### 1. 最终预测结果

```
比赛：{球队A} vs {球队B}
赛事：2026 年 FIFA 世界杯
阶段：{检索结果}
时间：{当地时间 + 北京时间}
地点：{城市 + 球场}
信息状态：{已确认 / 暂未完全确认 / 假设性预测}

90 分钟胜负倾向：{球队名胜 / 平局}
90 分钟判断：{主胜 / 平 / 客胜}
晋级判断：{仅淘汰赛输出}
独立模型倾向：{不看盘口时的方向 + 一句话理由}
盘口对照：{顺盘 / 逆盘 / 大小球 / 不碰 + 一句话理由}
投注主推：{让球盘 / 大小球 / 不碰}｜{具体方向}
```

## Known Pitfalls

- 不要只给胜平负，必须给盘口。
- 不要看到强队就自动让胜；先判断盘口是否已经把优势吃干净。
- 不要把热门媒体叙事当作盘口价值。
- 不要顺着盘口写总结。必须先做无盘口独立模型，再做盘口对照。
- 不要为了显得专业而永远正路。足球低比分、高波动，逆路要被认真建模。
- 如果最终仍然顺盘，必须写清为什么这次顺盘不是被市场牵着走。
- 不要用「看好某队」承载投注建议。必须拆成「90分钟胜负倾向」和「主推盘口类型」。
- 如果主推大小球，海报最大字必须是「主推：大/小 X.X」，不是球队名。
- 如果主推让球盘，必须写赢盘条件，避免用户把胜负倾向误读成让球推荐。
- 如果主推让半球、平半、半一、一球等亚洲盘，必须把赢盘、输盘、走水、赢半/输半条件说清楚。
- 不要假设用户理解 `-0.25`、`-0.5`、`+0.75`；投注建议必须翻译成人话。
- 不要只写「不追」「升盘不追」「降盘不追」。必须写「盘口失效条件」，并说明这是原推荐作废，不是改买反方向。
- 不要在同一条主推后又临时给第二个方向。盘口失效后只能写「需要重新判断」，不能写成隐藏备选投注。
- 不要因为怀疑诱盘就不敢下结论；疑似诱盘要写清触发证据。
- 第二张分析长图必须保留 `huxiang-card` footer：logo + 虎小象。
- 不要把内部回溯标签展示给球迷。`score_too_conservative` 这类词只能留在内部账本，输出时改写成「赛前证据校准」「风险提醒」。
- 不要只列天气数据。必须写它对节奏、体能、射门、门将和大小球的影响。
- 不要只说「阵容完整」。查得到确认首发时，必须列关键首发并说明如何影响盘口。
- 不要把波胆写成主推。波胆命中率天然低，只能小额可选；主推单位必须明显大于波胆合计。
- 不要为了显得覆盖全面而推荐 3 个以上波胆比分。三个预测比分是比赛路径，不等于三个下注比分。
- 不要输出具体金额，除非用户明确给了预算或单位金额。默认用 `U` 单位，且任何建议都必须限定在娱乐预算内。
- 不要把「最优投注建议」理解成必须下注。没有价格优势、比分分散、盘口已坏时，最优建议就是不投注或只看球。
- 不要生成普通信息卡风格海报。图 A 必须有毛笔字/手写冲击字、英文功能小字和强对抗构图，且主推盘口最大。
- 不要让图 A 只有文字和背景球场。必须同时有两队当家球星/代表性球员、国家视觉符号和队服/国旗色对抗；例如荷兰要想到狮子，日本要想到红日，这类符号由 Agent 14 每场先定。
- 不要接受普通黑体标题冒充毛笔字。图 A 顶部主标题必须有书写感、笔触感或体育战报冲击感；如果不达标，减少文案后重生。
- 不要在图 B 里临时自造字体和排版。必须先继承已验证母版，尤其是 `KingHwa_OldSong`（京華老宋体）字体路径；否则长图会变成普通 dashboard，不符合本技能。
