# Caption direction for story videos

字幕是画面的编辑结构，不是自动转录的装饰层。短片里最重要的判断、年代和转折，应该由字幕帮助观众“看见”。

## 1. Three caption modes

### `narration`

承担连续阅读。视觉应安静、稳定、少底板，避免与人物和道具争抢。每行通常 4–12 个汉字，最多两行；按语义短语切分，不按固定字数硬切。

### `thesis`

默认承担 5–8 个叙事铰链，例如核心命题、结构变化、因果转折和结尾追问。使用明显的字号差、关键词、空间位置和语义动画，但单次只强调一个思想。

当用户明确把字幕指定为全片最突出的视觉层时，可切换到“字幕主导模式”：通常可使用约 10–16 个英雄段落，但数量只由叙事铰链决定。把字幕设计成日期档案、制度图解、危机叠压、纪念碑、追问等不同类型的叙事海报，而不是重复同一套大字模板。中英文可以组成主标题与元数据的双层系统，但英文不逐句复述中文。

### `date_event`

承担年代、地点和事件锚点。可以使用竖排年号、档案标签、印章、刻度或纸签；年代必须一眼可辨，不能和旁白正文混成同一层级。

默认模式下，英雄字幕不应超过全片时长的 35–45%。字幕主导模式可提高到约 55–85%，但必须留下至少 2–4 个安静窗口，并让每个英雄段落拥有不同的叙事任务、空间位置或运动逻辑；占比提高不等于整片持续吼叫。

## 2. Typography hierarchy

- 展示字优先宋体或有历史重量的衬线中文字体；正文优先清晰的黑体。先检查本机真实字体，不能凭名称假定已安装。
- 建议两级：小号上下文 + 大号判断/动作。不要同时使用三种以上字号和两种以上强调色。
- 每屏只允许一个视觉重音。关键词通常 2–6 个汉字；整段旁白不能全部放大。
- 避免厚重的整句矩形底板。需要承托时，使用局部纸条、半透明墨迹、边缘撕纸或窄色签。
- 保留 9:16 平台安全区；字幕不得遮住脸、手、工具、关键物件或 `ledgers/beats.json` 中的承诺行动。

## 3. Semantic motion

字幕运动必须把句意变成动作，而不是套模板：

- 集中、汇聚、控制：文字或线条向一个中心收束。
- 扩张、贸易、传播：词组沿路径分散、传递或盖章落点。
- 依赖、脆弱、断裂：稳定排版随后出现偏移、裂纹、断线或局部失色。
- 战争、灾荒、欠饷：词组逐层压迫画面空间，避免娱乐化弹跳。
- 年代转折：年份成为切口、门轴、刻度或档案翻页。
- 追问：先移除解释层，只留下问题和足够停顿。

允许的基本动作：纸张擦入、印章落下、线条牵引、折页、钉住、撕裂、墨迹显影。避免 TikTok 式弹跳、无意义缩放、频繁抖动、逐字卡拉 OK 和每句相同的飞入。

## 4. Timing

- `narration`：短语整体出现，进入 120–220 ms；朗读完成后立即退出或在语义停顿处退出。
- `thesis`：关键词先于或同步于重读音节落下；至少保留 0.8 秒完整可读状态。
- `date_event`：年份至少保持 1.0 秒清晰状态。
- 连续字幕之间保留明确状态切换；上一句必须 hard kill，不能残留到下一镜。
- 所有时点以最终旁白音频的词级时间为准，不按脚本字数推算。

## 5. Required artifacts

`caption-style.json` 至少记录：字体、字号层级、色板、边距、安全区、三种 mode 的布局和运动规则。

`ledgers/caption-safe-zones.json` 使用 `schemas/caption-safe-zones.schema.json`。每个英雄段落记录主体保护区；保护对象不能只写语义名称，还要有随时间变化的 `keyframes` 坐标。同一段落中若人物、动作和关键物件分散，必须建立多个保护区，不能用一个宽泛框代替。

`ledgers/caption-beats.json` 使用 2.3 schema，每项至少记录：

```json
{
  "id": "cap-07",
  "mode": "thesis",
  "start": 24.76,
  "end": 26.77,
  "zh": "天下的命令，被拧成一根线",
  "en": "ONE CENTER / ONE LINE",
  "semantic_motion": "six routes converge into one taut line",
  "visual_avoid": ["emperor_face", "official_actions"]
}
```

SRT 仍需交付，但 SRT 只承担可移植文本，不代表最终烧录字幕的视觉设计。

## 6. Final-master QA

1. 渲染前运行 `scripts/qa-caption-occlusion.mjs`，默认每 0.3 秒或更密采样 DOM 几何；只抽三帧不足以发现滑入穿过人脸。
2. 为每个 `thesis` 和 `date_event` 从最终 MP4 抽取进入、中点、退出帧。
3. 检查 2 秒静音观看时是否仍能判断层级和重点。
4. 检查手机尺寸下是否可读；不要只在桌面预览器判断。
5. 检查是否遮挡人物动作、年代物件或构图焦点。
6. 自动通过后仍要查看最终母版证据帧，补上遗漏或运动中的主体保护区并重跑。
7. 检查文字动画是否表达句意；若换成另一句也成立，说明动作过于模板化。
8. 默认模式检查英雄字幕是否真正稀缺；字幕主导模式检查英雄段落是否足够多样，并确认安静窗口仍然存在。

报告必须区分 `intersectionAreaThresholdPx2` 与 `maxObservedIntersectionAreaPx2`；不得把配置阈值写成实测最大值。DOM 检查是渲染前门禁，不代表最终像素已经目视通过。

只有同步、可读、层级、避让和语义动作都通过，才可称为“字幕设计完成”。
